Execution Flow How /ralphtemplatetest orchestrates test-first development
Phase 1
Challenger & Proxy Negotiate
Challenger raises 5+ ambiguities. Proxy answers from codebase/docs.
If Proxy <75% certain, delegates to Researcher. Loops until satisfied.
Phase 1.5 — TEST-FIRST
Tester Creates Tests in Sandbox
Tester researches expected behavior, writes tests in /tmp/ralph-test-sandbox-SESSION_ID/
Tests verify BEHAVIOR not implementation details. Builder has NOT seen test code yet.
Tests are designed to fail initially (nothing implemented yet).
Phase 2
Builder Implements Against Tests
Builder implements iteratively. Cannot see test code before starting.
Challenger reviews after each milestone. Researcher consulted when uncertain.
Phase 3
Verify & Output Passphrase
All project tests pass. Challenger does final review. Builder outputs passphrase.
Phase 3.5 — SANDBOX EXECUTION
Run Tester's Tests in Sandbox
After passphrase is output, sandbox tests execute against the implementation.
Tests pass: promise kept, sandbox cleaned up.
Tests fail: promise REVOKED, Builder forced back to Phase 2.
Sandbox Architecture Tests live outside the project, cleaned up after execution
1. Create Sandbox
Directory: /tmp/ralph-test-sandbox-SESSION_ID/
Session-scoped to avoid multi-terminal collisions
Tests verify expected BEHAVIOR only
Independent of implementation details
→
2. Execute Tests
Runs after Builder outputs completion passphrase
Tests target the project's implementation
Pass: promise stays valid, proceed to cleanup
Fail: promise revoked, Builder loops back
→
3. Cleanup
trap "rm -rf /tmp/ralph-test-sandbox-*" EXIT
/tmp is ephemeral (appropriate for per-session tests)
Tests regenerated each session (not persistent)
No test code pollutes the project tree
TESTINGOFF Toggle How /ralphtemplate handles TESTINGOFF in arguments
Testing ON (default)
  • Role 5 (Tester) in prompt
  • Phase 1.5 (test creation)
  • Sandbox directory setup
  • Post-completion test execution
  • Promise revocation on test failure
  • Sandbox cleanup via trap
  • Command: /ralphtemplatetest "task"
TESTINGOFF
Testing OFF
  • Role 5 (Tester) stripped
  • Phase 1.5 stripped
  • Sandbox directory stripped
  • Post-completion tests stripped
  • Promise revocation stripped
  • Sandbox cleanup stripped
  • Falls back to 4-role /ralphtemplate behavior
ℹ
TESTINGOFF is detected as a standalone word in arguments. Unusual ALL-CAPS compound word avoids false positives. When present, the generated prompt omits all testing sections and produces a standard 4-role prompt identical to /ralphtemplate. Two separate command files: ~/.claude/commands/ralphtemplate.md (4 roles) and ~/.claude/commands/ralphtemplatetest.md (5 roles). Both are GLOBAL across all projects (located in ~/.claude/commands/).
