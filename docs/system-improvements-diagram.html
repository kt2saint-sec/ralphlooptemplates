Critical Fixes: Problems Discovered and Resolved
Critical Promise Detection Broken
Original plugin uses <promise> XML tags for completion detection via Perl regex
Claude Code strips XML tags from output before writing to transcript
Replaced with passphrase system: WORD NNNN WORD NNNN WORD NNNN (~8 trillion combos)
Detection uses grep -Fx exact line match — no XML at any point
Decisions 1, 7 — 18 tests verify zero false positives
Critical Plugin Updates Overwrite Patches
/plugin update runs git pull on marketplace directory, overwriting all patches
installPath in installed_plugins.json is IGNORED by Claude Code
cache-sync.sh syncs 6 files to 3 directories (marketplace PRIMARY)
cache-watchdog.sh detects mismatches on SessionStart
Decisions 14, 40, 42, 44 — 7 watchdog tests
Critical Session ID Mismatch
Original uses PPID — differs between setup script and stop hook processes
CLAUDE_SESSION_ID env var does not exist (verified)
Hook JSON session_id used for O(1) state file lookup
uuidgen fallback + flock-protected rename on first iteration
Decisions 2, 8, 9, 15, 16 — 13 rename migration tests
Discovery Hook Script Content Cached
Claude Code caches hook CONTENT in memory at session start (undocumented)
Editing hook files mid-session has zero effect on running hooks
Established "sync then new session" workflow
Verified by adding debug logging to all 3 copies — no output
Decision 41 — Empirically confirmed, not in official docs
Major State File Format Mismatch
Original plugin: ralph-loop.local.md (no session ID)
Our version: ralph-loop.{SESSION_ID}.local.md — glob mismatch
Dual glob pattern supports both formats
nullglob char-class trick: loca[l].md forces glob behavior
Decisions 24, 27, 30 — literal paths bypass nullglob
Major Frontmatter AWK Corruption
Original awk skips ALL --- lines in document, corrupting markdown prompts
Fixed: only skips first two --- (frontmatter delimiters)
