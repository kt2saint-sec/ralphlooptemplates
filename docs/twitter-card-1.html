CORE 5-Role Prompt Architecture Generated by ~/.claude/commands/ralphtemplatetest.md (TESTINGOFF strips Role 5)
Role 1: Challenger
  • Boris Antagonist Method
  • 5+ ambiguities identified
  • 2-3 approaches with tradeoffs
  • Routes questions to Proxy
  • Reviews after every milestone
  • Can force redesign
  • Checks naming + duplicates
Role 5: Tester v2
  • Activates Phase 1.5 (after Challenger)
  • Creates tests BEFORE Builder
  • Tests verify BEHAVIOR only
  • Sandbox: /mnt/nvme-fast/claude-workspace/sandbox/ralph-test-sandbox-ID/
  • Builder has NOT seen test code
  • Positive + negative + edge cases
  • trap cleanup on EXIT
Role 2: Builder
  • Implements iteratively
  • Max 10 iterations before escalate
  • Logs: status, root cause, fix, result
  • <75% certain: delegates to Researcher
  • Must NOT code before Tester
  • Fixes code, not tests on failure
Role 3: Proxy
  • Human stand-in (never says "ask user")
  • Reads CLAUDE.md, LEARNINGS.md, patterns
  • Confidence % on each answer
  • <70% flagged REVIEWABLE
  • <75% delegates to Researcher
  • Always decides, never defers
Role 4: Researcher
  • Independent, unbiased fact-finder
  • Activates ONLY on delegation
  • Subagents + web + MCPs
  • context7, fetch, brave, github
  • Output: Question, Sources, Findings
  • Confidence %, Caveats
