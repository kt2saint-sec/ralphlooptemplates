What was broken (before)
  • <promise> XML tags stripped by Claude Code rendering — detection never fires
  • PPID differs between setup and hook — session ID mismatch
  • AWK skips all --- lines — corrupts markdown prompts
  • Simple completion words cause false-positive exits
  • No research step — Builder guesses and wastes iterations
  • Plugin updates silently overwrite all fixes (cache-sync workaround)
  • No multi-terminal session isolation
  • Hook edits mid-session have no effect (undocumented caching)
What works now (after)
  • Passphrase system (~8T combos) with grep -Fx plain-text matching
  • Hook JSON session_id with uuidgen fallback + flock rename
  • Fixed AWK: only skips first two --- (frontmatter only)
  • Auto-generated passphrase eliminates false positives
  • Researcher role (Role 4) activates at <75% certainty
  • Hybrid architecture: settings.json hooks + local commands (session 12)
  • /plugin update cannot affect hooks or commands (overwrite solved)
  • Session-scoped state files prevent cross-terminal contamination
  • Workflow simplified: edit → new session (no cache-sync)
  • /ralphtemplatetest: 5-role system with Tester + sandbox (session 12)
